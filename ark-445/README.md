# ARK-445: Tri-State Authorization Discrimination (ALLOW / HOLD / DENY)

**ARK-445** is a preregistered metrological test of whether the verify-then-execute (VBE) authorization boundary can discriminate **three** distinct states — ALLOW (execute with high confidence), HOLD (ambiguous/pending), and DENY (refuse with high confidence) — rather than only binary ALLOW/DENY. The experiment uses 10 arms on a 2-qubit system (authorizer Q_A, payload Q_P), encoding ALLOW as A=|1⟩ (expect P fires, S_A ≥ 0.90), DENY as A=|0⟩ (expect P refuses, L_D ≤ 0.02 corrected), and HOLD as A=|+⟩ or |−⟩ (expect intermediate P(execute) ∈ [0.40, 0.60] due to measurement-induced probabilistic collapse), with preregistered thresholds requiring tri-state discrimination margin Δ_H ≥ 0.30 and an in-situ SPAM kill-gate (≤ 0.02 per qubit). A PASS demonstrates that three metrologically separable authorization outcomes are accessible on this setup — not new physics (superposition → probabilistic measurement is textbook quantum mechanics), not a cryptographic guarantee, and not generalizable beyond the selected qubits/backend. Full preregistration (`ARK_445_preregistration.md`) and code committed with SHA-256 hashes before any job submission; commit hash acts as the LOCK (recorded in `RUN_LOG.md`). Extends ARK-441/446/442/444/443 from binary to tri-state decision spaces.
