WHY THIS ADDENDUM EXISTS
----------------------------------------------------------------
Three post-lock changes are disclosed here, following the ARK-447
v1.0->v1.1 full-disclosure precedent. The raw hardware counts, job
IDs, SPAM gate, and the PASS verdict are UNCHANGED by all of them.

1. POST-DATA ANALYSIS CORRECTION (v1.2) — circuits/ark_449_analysis.py
   The v1.1 analysis reported SPAM-corrected ALLOW retention as the
   PRIMARY S_A. That correction divided the payload P(Q_P=1) by
   (1 - 2*SPAM_A), i.e. it applied the AUTHORIZER qubit readout error
   (SPAM_A) to the PAYLOAD outcome. Because the ALLOW quantity is the
   payload bit, that correction lacks a valid payload-readout-assignment
   model. v1.2 makes the RAW payload P(Q_P=1) the PRIMARY S_A metric and
   demotes the SPAM-corrected values to secondary/descriptive only.
   Verdict unchanged (raw): S_A_min=0.9794, L_D_max=0.0011, Delta_B=0.9783.

2. ADMINISTRATIVE LABEL FIXES — ARK_449_preregistration.md
   Residual "ark-449-v1.0-lock" / "Version 1.0" label references (in the
   lock-instruction, strict-execution-sequence, and MANIFEST-placeholder
   sections) were corrected to v1.1 to match the actually-executed lock.
   No hypotheses, arms, criteria, thresholds, or metric definitions were
   changed.

3. HARDWARE-SUBMISSION API FIXES — circuits/ark_449_circuit.py
   Pre-data fixes required to run on current qiskit-ibm-runtime:
   SamplerV2 'backend=' -> 'mode=', removal of the unsupported context
   manager, and circuit-width-aware transpile layout. These were applied
   before any results were read.
