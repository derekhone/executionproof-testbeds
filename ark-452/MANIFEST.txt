This MANIFEST is committed and SHA-256-hashed before any hardware

Pre-lock drafting correction (no lock, commit, or hardware data
existed before this correction):
  1. ALLOW execution gate H -> X. A Hadamard would have put the
     payload in a 50/50 superposition (P(Q_P=1) ~ 0.50), which could
     never meet C1 (S_A_min >= 0.90) — a guaranteed FAIL by
     construction. Corrected to drive the payload to |1> via X so
     ALLOW retention can pass. The SPAM_P circuit retains H by design
     (it needs the |+> symmetry diagnostic).
  Post-correction noiseless dry-run: ALLOW arms (1, 8) = 1.0000,
  all seven DENY arms = 0.0000, SPAM gate passed. No criteria,
  hypotheses, arms, or thresholds were changed.
  See ARK_452_preregistration.md top notice for full disclosure.

Analysis doctrine: RAW-PRIMARY. ARK-452 uses one payload qubit only,
so there is no authorizer-qubit SPAM_A correction and none is applied.
Raw P(Q_P=1) is the primary metric for S_A and L_D. SPAM_P gates the
run and is never subtracted from L_D. This deliberately avoids the
ARK-449 v1.1 error (corrected in ARK-449 v1.2) of applying an
authorizer-qubit readout correction to a payload outcome.
